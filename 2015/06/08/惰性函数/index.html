惰性函数

发表于 2015-06-08

惰性函数表示函数执行的分支只会在函数第一次掉用的时候执行，在第一次调用过程中，该函数会被覆盖为另一个按照合适方式执行的函数，这样任何对原函数的调用就不用再经过执行的分支了。这样做的目的就是提高函数的运行效率，举个例子：
我们知道IE6~8不支持XMLHttpRequest()方法，支持ActiveXObject(‘Microsoft.XMLHTTP’)。因此在创造一个AJAX请求，必须写一个兼容方法，如下：

var getXHR = function(){
    if(window.XMLHttpRequest) {
         return new XMLHttpRequest();
    }
    return new ActiveXObject('Microsoft.XMLHTTP');
};

如果页面多次执行这个getXHR方法，每次都需要判断下浏览器是否有没有XMLHttpRequest这个属性，但是其实第一次运行这个方法时，我们就已经知道浏览器是否支持该属性了，所以我们接下去完全没有必要再继续做判断，有没有一种逻辑可以实现样的方法呢？答案是肯定的，这便是我们的惰性函数，代码如下：

var getXHR = (function(){
if(window.XMLHttpRequest) {
    return function(){
        return new XMLHttpRequest();
    }
}
return function(){
    return new ActiveXObject('Microsoft.XMLHTTP');
}
})();

如上所示，自制行函数运行后，就可以得知浏览器是否有没有XMLHttpRequest这个属性，并根据结果重写了getXHR方法，这样接下去每次只需getXHR方法就不用继续做判断了。
