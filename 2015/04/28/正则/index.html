正则的规则

正则用来处理字符串的规则。

  • 正则元字符(在两个斜杠之间包含的字符都是正则都是正则的元字符)：

    1. 具有特殊意义的元字符
      \d -> 匹配一个0-9的数字,相当于[0-9],和它相反的是\D ->匹配一个除0-9的任意字符
      \w -> 匹配一个0-9、a-z、A-Z、的数字或字符,相当于[0-9a-zA-Z]
      \s -> 匹配一个空白字符(空格、制表符…)
      \b -> 匹配一个单词的边界
      \t -> 匹配一个制表符
      \n -> 匹配一个换行
      . -> 匹配一个除了\r\n以外的任意字符
      ^ -> 以某一个元字符开头
      $ -> 以某一个元字符结尾
      \ -> 转义字符
      x|y -> x或者y的一个
      [xyz] -> x、y、z中的任意一个
      [^xyz] -> 除了xyz中的任意一个字符
      [a-z] -> 匹配a-z中的任意一个字符
      [^a-z] -> 匹配除了a-z中的任意一个字符
      () -> 正则中的分组
    2. 代表出现次数的量词元字符
      • -> 0到多个
      • -> 1到多个
        ? -> 0到1个
        {n} -> 出现n次
        {n,} -> 出现n到多次
        {n,m} -> 出现n到m次
    3. 代表本身意思的元字符：除了以上的,在字面量方式中,我们出现的其他任意字符代表的都是自己本身的意思。

  • 正则一些细节点

    1. 关于[ ]
      • [+] ->中括号中出现的单字符（+，-，*，！等等）都代表的是本身的意思,
      • [12-65] ->这个不是12-65而是1/2-6/5三者中的一个

    2. ()分组：把一个大正则划分成几个小正则

      • 分组的第一个作用:改变默认的优先级
        var reg = /^18|29$/; //->18、19、182、189、829、1829...这些都符合,不是我们想要的只是18或者29
var reg = /^(18|29)$/;//->现在只能是18或者29了
      • 分组的第二个作用:分组引用,如\1代表的是和和第一个分组出现一模一样的内容
        var reg = /^\w(\w)\1\w$/;//->food week door wood good...

      • 分组捕获:不仅大正则匹配的内容捕获到,而且还可以把每一个小分组单独匹配的内容也捕获到

        //身份证
//->13(省) 0828(市和县) 1990(年) 12(月) 04(日) 06 1(奇数男 偶数女) 7(数字或者X)
var reg = /^([1-9]\d)(\d{4})(\d{4})(\d{2})(\d{2})\d{2}(\d)(?:\d|X)$/;
console.dir(reg.exec("130828199012040617"));
console.log("130828199012040617".match(reg));

//->exec：把正则匹配到的结果捕获到(包含小分组匹配的内容)
//->得到的内容首先是一个数组
//->0:"130828199012040617" ->大正则匹配的内容
//->1:"13" ->第一个分组匹配的内容
//->2:"0828" ->第二个分组匹配的内容
//->3:"1990" ->第三个分组匹配的内容
//->6:"1" ->第六个分组匹配的内容
//->7:"7" ->第七个分组匹配的内容  这个内容对于我们没有用 我们这样写(?:\d|X) ->?:在分
组中的意思是：只进行规则的匹配,但是捕获的时候不进行捕获

//->index: 0 ->大正则捕获的位置在字符串中的索引
//->input: "130828199012040617" ->捕获的原始字符串
    3. 关于?的几种情况
      • 放在非量词元字符后面,代表出现0-1次
      • 放在量词元字符后面,代表取消捕获时候的贪婪性,例如:reg=/\d+/; reg.exec(“2014”) -> “2014” 但是如果正则这样写 reg=/\d+?/; reg.exec(“2014”) -> “2”
      • 在分组开头加?:,代表当前的分组只是匹配不进行捕获,例如:/^(?:\d+)$/
      • 在分组开头加?=,正向预查,例如:/^can(?=1|2)$/ 只有”can1”和”can2”才符合
      • 在分组开头加?!,负向预查,例如:/^can(?!1|2)$/ 除了”can1”和”can2”不符合,其他的只要是”can(任何的东西)”都符合
        正则的方法

  • 正则捕获（exec）

    1. 我们使用exec进行正则的捕获:首先判断字符串和正则是否匹配，不匹配的话，捕获到的结果为null，匹配的话,把和正则匹配的内容都捕获到。

    2. 正则的捕获是存在懒惰性的:一个字符串中有多个都和正则匹配,但是我们执行一次exec只能捕获到第一个匹配的,再次执行exec还是只能捕获到和第一个匹配的…

      //->reg.lastIndex:上一次正则捕获的结束位置/下一次捕获的起始查找位置
//->懒惰性:每一次执行exec后并没有改变它的lastIndex的值，但是即使我们自己通过reg.lastIndex = 11把其修改了,发现最后的结果并没有改变；

str = "can2014sw2015";
console.log(reg.lastIndex);//0
console.log(reg.exec(str));//["2014", index: 7, input:...]

console.log(reg.lastIndex);//->7
console.log(reg.exec(str));//->["2015", index: 9, input: "can2014sw2015"]

    3. 解决正则的懒惰性->给正则增加修饰符g
      正则的修饰符:i(ignoreCase 忽略字母的大小写)、m(multiline 多行匹配)、g(global 全局匹配)

      var str1 = "b c d";
var reg1 = /(^| )b( |$)/g;
console.log(reg1.exec(str1));  //["b ", "", " ", index: 0, input: "b c d"]
console.log(reg1.lastIndex);   //2
console.log(reg1.exec(str1));  //null,往下匹配没有找到b,返回null
console.log(reg1.lastIndex);   //0

var reg2 = /(^| )b( |$)/;
console.log(reg2.exec(str1));  //["b ", "", " ", index: 0, input: "b c d"]
console.log(reg2.lastIndex);   //0
/*********************************************************/

//->即使我们把正则的懒惰解决了,但是想要把所有匹配的内容都得到,我们还需要执行多次exec才可以,
这样的话我们自己编写一个while循环,不管有多少匹配的,最后都得到
reg = /\d+/g;
str = "str2014can2015foo2016";
var ary = [];
var res = reg.exec(str);
while (res) {
   ary.push(res[0]);
   res = reg.exec(str);
}
console.log(ary);

//在字符串中提供了一个叫做match的方法,可以执行一次就把所有匹配的内容捕获到
reg = /\d+/g;
str = "str2014can2015foo2016";
console.log(str.match(reg));//->["2014", "2015", "2016"]

    4. 正则的贪婪性:每一次捕获的时候,是按照匹配的最长结果进行捕获的

      reg = /\d+/g;
str = "can2014";
console.log(reg.exec(str));//->["2014"...]

//解决办法:在量词元字符后面,添加一个?即可
reg = /\d+?/g;
str = "can2015";
console.log(reg.exec(str));//->["2"...]
//注意 match的局限性:在分组捕获中,match不能捕获到小分组匹配的内容

    5. 正则与replace
      replace的第二个参数为函数，表示以这个函数的运行结果为替换值，当这个函数运行的时候,replace方法会自动给这个函数最少传三个参数，分别是1、前边正则匹配到的结果；2、此结果在原字符串中的索引位置；3、原输入字符串，如果正则有n个分组的话，参数就有3+n个，其中正数的第一个参数为result，倒数第二个是index，最后一个是input参数。

      var str4 = "123dhue34r556de";
//第二个参数开始就是分组内容（第一个分组）
var str5 = str4.replace(/(\d+)/g,function(result,result1){ 
    return eval(result1.split("").join("+"));
})
console.log(str4);//6dhue7r16de
